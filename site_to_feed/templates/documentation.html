Step 1. Specify a source page address (URL)

This step retrieves the HTML source from a given (URL) and is required for step 2.

Step 2. Define extraction rules

This step defines the pattern to extract from the HTML retrieved in step 1.

A general understanding of HTML elements and the </a> (anchor element) is recommended.

Global Search Pattern

The goal of this search pattern is to define which part of the HTML document you want to parse.

It's generally recommended to use {%} to select the entire HTML source.

Item Search Pattern

The goal of this search pattern is to retrieve the title, hyperlink (URL), and content.

Patterns generally contain the following elements:

  • <> – The opening tag of an HTML element
  • </> – The closing tag of an HTML element that matches the opening tag with an additional / before the element name
  • {%} – Defines the element to assign the title, hyperlink, or content to
  • {*} – Indicates that the HTML between the previous element and the next element can be ignored by the pattern

A properly formatted pattern should have three instances of {%} for declaring the title, hyperlink, and content. It should also have three instances of {*} for declaring which parts of the HTML to ignore between the initial opening tag <> and the final closing </> tag.

Example

Suppose a blog has multiple posts in the following format:

<article>
  <a title="Post 1" href="https://www.example.com/blog/Post_1">
    Post 1
  </a>
  <p>
    Content of the blog post.
  </p>
</article>

The Global Search Pattern could be:

{%}

The Item Search Pattern could be:

<article>{*}
title="{%}"{*}
href="{%}"{*}
<p>{%}</p>

This pattern would look for HTML elements in the <article> element and assign the title to {%1}, the hyperlink to {%2}, and the content to {%3}. The assigned items are numbered in the order they are matched in the pattern, and the numbering is used for step 3.
